## Fix Phase 2 DGP Validation Bug

**Context:** User reported that Phase 2 simulations (DGP7-9) didn't run on O2. Investigation revealed validation error in batch script.

**Problem:**
- Phase 2 jobs immediately failed: "Invalid DGP: dgp9 (must be dgp1, dgp2, dgp3, dgp4, dgp5, or dgp6)"
- Root cause: `run_batch_replications.R` line 47 hardcoded validation only allowed dgp1-6
- The code to handle dgp7-9 existed (lines 111-122), but was unreachable due to early validation

**Investigation:**
```bash
# O2 error log showed:
cat logs/dml_batch_34803901_4.err
Error: Invalid DGP: dgp9 (must be dgp1, dgp2, dgp3, dgp4, dgp5, or dgp6)
Execution halted
```

**Root Cause Analysis:**
1. ✓ DGP functions exist: `generate_dgp7`, `generate_dgp8`, `generate_dgp9` in `dgps_phase2.R`
2. ✓ Function aliases exist: `generate_dgp7 <- generate_dgp_deep_interaction`
3. ✓ Code to call them exists: lines 111-122 in `run_batch_replications.R`
4. ✗ Validation blocked them: line 47 only allowed dgp1-6

**Solution Implemented:**

**File:** `simulations/run_batch_replications.R`

1. **Updated help text (line 17):**
   ```r
   # Before: "DGP name: dgp1, dgp2, or dgp3"
   # After:  "DGP name: dgp1 through dgp9"
   ```

2. **Updated validation (lines 44-48):**
   ```r
   # Before:
   if (!opt$dgp %in% c("dgp1", "dgp2", "dgp3", "dgp4", "dgp5", "dgp6"))

   # After:
   if (!opt$dgp %in% c("dgp1", "dgp2", "dgp3", "dgp4", "dgp5", "dgp6", "dgp7", "dgp8", "dgp9"))
   ```

**Verification:**

Created `test_phase2_batch_fix.R` to verify:
- ✓ Function aliases exist
- ✓ DGP7 generates binary outcome, tau = 0.10
- ✓ DGP8 generates continuous outcome, tau = 0.10
- ✓ DGP9 generates binary outcome, tau = 0.10, weak overlap (53%)

All tests passed ✓

**Why This Happened:**
Validation was added when only DGP1-3 existed, updated for DGP4-6, but not updated again when Phase 2 DGPs were designed. The code to handle dgp7-9 was added but the gatekeeper validation wasn't updated.

**Lesson:**
When adding new DGPs, search for validation code:
```bash
grep -n "dgp1.*dgp" run_batch_replications.R
```
