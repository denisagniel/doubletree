## Fix DGP5 Continuous Outcome Bug

**Context:** Phase 1 simulation analysis revealed critical bug preventing forest and linear baseline methods from running on DGP5 (continuous outcomes).

**Problem:**
- 6,000 failures (100% failure rate for forest/linear on DGP5)
- Error: "Y must be binary (0/1)"
- Root cause: Hardcoded binary outcome assumption in method_forest.R (line 63-64) and method_linear.R (line 60-62)
- Tree method works fine (doubletree handles continuous outcomes correctly)

**Solution Implemented:**

1. **method_forest.R** - Added continuous outcome support:
   - Auto-detect outcome type: binary vs continuous
   - Binary: ranger probability forest (probability=TRUE)
   - Continuous: ranger regression forest (probability=FALSE)
   - Updated propensity clipping: [1e-6, 1-1e-6] → [0.01, 0.99]

2. **method_linear.R** - Added continuous outcome support:
   - Auto-detect outcome type: binary vs continuous
   - Binary: glm(family=binomial) for logistic regression
   - Continuous: lm() for linear regression
   - Updated propensity clipping: [1e-6, 1-1e-6] → [0.01, 0.99]

3. **test_continuous_outcome_fix.R** - Verification script:
   - Tests both methods on DGP5 (n=400, τ=0.10)
   - Forest: θ̂ = 0.129, 95% CI [0.039, 0.218], covers τ=0.10 ✓
   - Linear: θ̂ = 0.109, 95% CI [0.071, 0.146], covers τ=0.10 ✓

4. **slurm/resubmit_dgp5_forest_linear.sh** - O2 resubmission script:
   - Resubmits ONLY DGP5 forest and linear jobs
   - 6,000 replications (3k forest + 3k linear)
   - 14 array jobs, expected runtime: 4-6 hours

## Phase 2 DGP Design (Afternoon)

**Context:** Reviewed Phase 1 results and identified gap in simulation coverage.

**Problem Identified:**
- Phase 1 (DGPs 1-6): All have linear/simple nuisances → favor linear methods
- Tree: Good coverage (96.5%) but worse RMSE (1-50x penalty)
- **Critical gap:** No demonstration of tree's value proposition
- Cannot answer: "Why use trees instead of flexible GLM with interactions?"

**Solution: Create Phase 2 DGPs Where Tree Wins**

### DGP7: Deep 3-way Interaction
- Propensity: e(X) has 2-way interaction X1*X2
- Outcome: m0(X) has 3-way interaction X1*X2*X3
- **Result:** Tree RMSE = 0.0104 vs Linear = 0.0151 (tree wins 0.69x) ✓

### DGP8: Double Nonlinearity (Key Innovation!)
**Breakthrough:** Must break double robustness by misspecifying BOTH e(X) AND m0(X)

**Key Insight:**
- EIF-based ATT is doubly robust: tolerates misspecification of e(X) OR m0(X)
- Phase 1 had linear e(X) → double robustness protected linear
- Early attempts (quadratics) → linear still approximates reasonably
- **Solution:** Sin/cos functions in BOTH models
  - Propensity: e(X) ~ sin(2πX1) + cos(2πX2)
  - Outcome: m0(X) ~ sin(2πX1) + cos(2πX2) + X1*X2
  - Linear cannot approximate trigonometric functions
  - Tree's piecewise approximation works well

**Result:** Tree RMSE = 0.1254 vs Linear = 0.2055 (tree wins 0.61x) ✓
- **Linear fails to cover truth!** (✗)
- Tree maintains coverage (✓)

### DGP9: Weak Overlap (Stress Test)
- Propensity: e(X) = plogis(-4 + 8*X1) (very steep)
- Creates near-deterministic treatment (only 55% have e ∈ [0.1, 0.9])
- **Result:** Linear wins RMSE (expected), but both maintain coverage ✓
