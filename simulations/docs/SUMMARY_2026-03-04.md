## Issue 2: DGP Misspecification (Discovered & Fixed Today)

### The Problem

**Original DGPs** (dgps_realistic.R, dgps_final.R, dgps_continuous.R) defined:

```r
p0 <- plogis(-0.5 + 0.4 * X1 + 0.3 * X2)
p1 <- plogis(-0.5 + 0.4 * X1 + 0.3 * X2 + tau)  # tau on LOGIT scale
```

- Additive shift `tau` on **logit scale**
- Does NOT translate to ATT = tau on probability scale
- For tau=0.15: true ATT ≈ 0.037 (75% smaller!)

### Why This Matters

For binary outcomes Y ~ Bernoulli(p):
- ATT = E[p1 - p0 | A=1] (difference in **probabilities**)
- Logit-scale: plogis(logit(p) + τ) ≠ p + τ
- The relationship is nonlinear due to the logistic transformation

### The Solution

**Corrected DGPs** (dgps_att_correct.R) use additive effects on **probability scale**:

```r
p0 <- plogis(-0.3 + 0.5 * X1 + 0.4 * X2)
p1 <- pmin(p0 + tau, 1)  # Direct additive effect, capped at 1
```

- Now E[p1 - p0] = tau exactly
- True ATT matches specified tau
- Valid inference achieved

### Future DGP Design

**Key lesson:** For binary outcomes, additive effects must be on **probability scale**, not logit scale.

**Template for ATT-controlled DGPs:**
```r
# Baseline outcome (logistic regression)
p0 <- plogis(linear_predictor)

# Treatment effect (additive on probability scale)
p1 <- pmin(p0 + tau, 1)  # Cap at 1

# Now: E[Y1 - Y0 | A=1] ≈ tau
```

**Alternative:** For continuous outcomes, use additive effects directly:
```r
Y0 <- mu0 + epsilon
Y1 <- mu0 + tau + epsilon  # ATT = tau exactly
```

## Technical Notes

### Discretization
- TreeFARMR's `fit_tree()` automatically discretizes continuous features
- Use `discretize_method = "quantiles"`, `discretize_bins = "adaptive"`
- For n=400: ~2 bins/feature → 8 binary indicators for 4 continuous features

- Use `capture.output(..., file = tempfile())` to redirect
- Not an issue for correctness, just noisy logs
